## Language Routing

`index.md` at the root detects the browser language (`navigator.language`) and redirects to `/intelligence/en/` or `/intelligence/ja/`. It contains no content.

Each post/page declares its language via the `lang:` front matter field. Section pages filter posts by `lang` using Liquid.

**Important**: `_config.yml` must keep `baseurl: /intelligence` set correctly. This value is used by Liquid to generate URLs in sitemaps and internal links. If baseurl is wrong, sitemap URLs will 404 and search engine indexing will break.

---

## Sitemaps

Sitemaps are split by language and generated by Liquid templates (not jekyll-sitemap, which is intentionally disabled in `_config.yml`):

- `en/sitemap.xml` — contains only pages/posts with `lang: en`
- `ja/sitemap.xml` — contains only pages/posts with `lang: ja`

The split structure exists because Google Search Console properties are registered per language directory, and each property can only accept sitemaps within its own scope. Do not consolidate these into a single root-level sitemap — it will break Search Console submission.

Posts without a `lang:` front matter field are excluded from both sitemaps by design.

## IndexNow Auto-Notification

On every push to `main`, `.github/workflows/indexnow.yml` runs and notifies Bing (and Yandex) of all URLs in both sitemaps. No manual action is required when publishing new posts — just push and the notification goes out automatically.

### Required files and secrets

- **API key file**: A `.txt` file named after the IndexNow API key is placed at the repository root. Its content is the key string itself. This file must be preserved — deleting it will break the notification endpoint verification.
- **GitHub Secret**: `INDEXNOW_KEY` is registered in repository settings (Settings → Secrets and variables → Actions). The workflow reads the key from this secret. The same key is shared between `icdomain.github.io` and `intelligence` repositories.

### How it works

1. Push to `main` triggers the workflow
2. Workflow parses all `.md` files in the repository, extracts `permalink` and `lang` from front matter, and builds the URL list directly — no live site fetch required
3. All URLs are sent as a single POST request to `https://api.indexnow.org/indexnow`
4. Success/failure is logged in the Actions run

If the workflow fails, check that `INDEXNOW_KEY` is still registered in Secrets and that the key file at the repository root is still accessible via HTTP.

## Post Format

New posts come in **pairs**: one `-en.md` and one `-ja.md` with the same slug.

```
_posts/YYYY-MM-DD-slug-en.md
_posts/YYYY-MM-DD-slug-ja.md
```

Use `_templates/post-ja.md` as the starting point and create both files. Key front matter:

```yaml
layout: post
title: ...
description: ...              # under 120 characters
permalink: /ja/archives/YYYY/MM/DD/slug/
lang: ja                      # or "en" for the English file
alt_lang_url: /en/archives/YYYY/MM/DD/slug/
date: YYYY-MM-DDTHH:MM:SSZ
last_modified_at: YYYY-MM-DDTHH:MM:SSZ
author: founder
categories: [CATEGORY]
tags: [tag1, tag2]
```

For the `-en.md` file, swap `lang: ja` → `lang: en`, and swap the `permalink` / `alt_lang_url` paths accordingly.

Note: `permalink` values are relative to the site root, and Jekyll automatically prepends `baseurl` (`/intelligence`) when resolving them. Do not hardcode `/intelligence/` into permalink values.

The `lang:` field is mandatory — posts without it will not appear in either sitemap and therefore will not be submitted to search engines.

## Adding New Content

1. Copy `_templates/post-ja.md` for posts, `_templates/page-ja.md` for section pages.
2. Create both `-en.md` and `-ja.md` files.
3. Fill in `title`, `description`, `permalink`, `alt_lang_url`, `date`, `categories`, `tags`.
4. Ensure `lang:` is set correctly in each file (`en` or `ja`).
5. Write the body in the appropriate language for each file.
6. Push to `main` — sitemaps will regenerate and IndexNow will notify Bing/Yandex automatically.
